# ImageRail / 画轨

这是一个给美术从业者使用的图片版本管理桌面应用 MVP。当前版本使用 Electron + HTML + CSS + JavaScript，没有引入复杂前端框架。

## 项目结构

```text
imagerail/
├─ package.json
├─ README.md
└─ src/
   ├─ main.js
   ├─ preload.js
   └─ renderer/
      ├─ index.html
      ├─ styles.css
      └─ renderer.js
```

## 每个文件做什么

- `package.json`：项目说明、启动命令、Electron 依赖。
- `src/main.js`：Electron 主进程。负责打开窗口、选择项目文件夹、读取和保存项目数据、复制图片、自动命名图片。
- `src/preload.js`：安全桥梁。让网页界面可以调用主进程提供的桌面能力。
- `src/renderer/index.html`：应用界面的 HTML 结构。
- `src/renderer/styles.css`：应用界面的样式，让它像横向时间线画板。
- `src/renderer/renderer.js`：界面交互逻辑。负责新建轨道、拖入图片、显示缩略图、编辑备注和状态、点击预览。

## 第一次运行前

先安装 Node.js。建议安装 LTS 版本：

https://nodejs.org/

安装完成后，重新打开终端，输入：

```bash
node -v
npm -v
```

如果都能显示版本号，说明安装好了。

## 安装依赖

在本项目文件夹里运行：

```bash
npm install
```

## 启动应用

在本项目文件夹里运行：

```bash
npm start
```

## 如何测试当前 MVP

1. 点击顶部的“选择项目文件夹”。
2. 选择或新建一个文件夹，例如 `D:\ImageRail-Test`。
3. 点击“新建轨道”，会出现“轨道 A”。
4. 从电脑文件夹里拖一张图片到轨道 A。
5. 应用会把图片复制到你选择的项目文件夹下的图片总文件夹，默认叫 `images`：

```text
项目文件夹/
└─ images/
   └─ track_A/
      └─ A_1.0.png
```

项目数据不会再放到你选择的项目文件夹里，而是放到应用所在目录：

```text
imagerail-1-0-1-1-1/
└─ project-data/
   └─ projects/
      └─ ImageRail-Test_xxxxxxxxxx.json
```

6. 再拖第二张图，会命名为 `A_1.1.png`。
7. 新建第二条轨道后，图片会进入 `track_B`，命名为 `B_1.0.png`、`B_1.1.png`。
8. 可以编辑备注和状态，应用会保存到应用目录下的项目数据 JSON。
9. 点击缩略图可以放大预览。
10. 点击“移除记录”只会从项目数据里删除记录，不会删除原图片文件。

## 重命名功能

- 顶部“重命名图片文件夹”：会重命名你选择的项目文件夹里的图片总文件夹。例如项目是 `E:\SFC\测试`，默认图片文件夹是 `E:\SFC\测试\images`，可以改成 `E:\SFC\测试\分镜头`。不会修改外层的 `测试` 文件夹。
- 轨道左侧“重命名轨道”：会重命名图片总文件夹里的轨道文件夹，并更新轨道内图片的路径记录。
- 轨道左侧或图片卡片里的“重命名前缀”：会修改同一条轨道里所有图片的真实文件名。例如把前缀 `A` 改成 `hero` 后，图片会变成 `hero_1.0.png`、`hero_1.1.png`、`hero_1.2.png`。
- 轨道左侧“删除轨道”：只删除应用里的轨道记录，不会删除硬盘里的轨道文件夹和图片。
